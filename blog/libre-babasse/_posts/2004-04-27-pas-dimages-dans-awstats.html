--- title: Pas d'images dans awstats ---

Je viens de trouver comment résoudre un petit problème que j'ai depuis quelque temps, et que je ne comprenais pas. Il semblerait qu'il y ait un petit manque dans le paquet debian pour awstats en testing. En effet, après installation par le classique apt-get install awstats, tout marchait bien avec apache, sauf qu'il n'y avait pas d'images, et pas de fond coloré aux barres de statistiques, ce qui en soit n'était pas critique, mais donnait quand même un profond sentiment d'échec au geek que je suis.

\r\n\r\n

Il s'avère que le problème vient d'un manque de configuration dans apache (et dans apache-ssl). Et pour le résoudre, il suffit donc d'ajouter cette configuration. L'emplacement exact des images est spécifié dans le fichier de configuration de awstats (sous Debian, c'est dans /etc/awstats/awstats.conf), mais je prend ici tout simplement la configuration par défaut du paquet. J'ai ajouté un fichier dans /etc/apache/conf.d (pour que cela marche, il faut donc une ligne Include /etc/apache/conf.f dans le fichier de configuration de apache), dont voici le contenu :

\r\n

\r\n\r\nAlias /awstats-icon /usr/share/awstats/icon\r\n<Directory /usr/share/awstats/icon>\r\nOptions FollowSymLinks\r\nAllowOverride Limit Options FileInfo\r\n</Directory>\r\n\r\n
