--- title: Montage de systèmes de fichiers en cascade ---

Le symptôme

\r\n

\r\nmount: must be superuser to use mount vous est renvoyé par votre système lorsque vous tappez mount /cdrom alors même que vous avez bien vérifié avoir ajouté l'option user sur la ligne correspondante à /cdrom dans le fichier /etc/fstab...\r\n

\r\n\r\n

Une solution possible

\r\n

\r\nIl est possible que vous ayez un peu joué (cf un peu plus loin) avec les permissions des commandes mount et umount. Si vous avez effectivement joué avec, il faut vous assurer que ces commandes sont bien setuid root. Vous pouvez par exemple utiliser la commande suivante : sudo chmod u+s /bin/{,u}mount.\r\n

\r\n\r\n

Mount, Samba, Superutilisateurs, ...

\r\n

\r\nEt donc, pour donner une explication sur le pourquoi du comment de mes tripatouillages avec la commande mount : j'avais un problème de partage de répertoire samba, de mobilité et de mots de passe en clair... En effet, un serveur samba partage chez moi un répertoire public et les répertoires utilisateurs de mon réseau. J'y accède depuis un mobile et depuis mon fixe. Je ne veux pas que\r\n

\r\n\r\n
    \r\n
  • les identifiants utilisateurs apparaissent dans le fichier /etc/fstab.
    • \r\n
  • les répertoires soient forcément montés automatiquement au démarrage sur le portable.
    • \r\n
\r\n\r\n

\r\nJe vous propose la solution suivante, et vous demande toute idée supplémentaire pour améliorer le tout. Sur le fixe (mono-utilisateur), c'est relativement simple, j'ai tout simplement déporté les identifiants dans un fichier de mon répertoire utilisateur et utilisé les lignes de configuration suivantes dans /etc/fstab :\r\n

\r\n\r\n

\r\n//pastis/homes /home/clark/server smbfs user,noauto,credentials=/home/clark/.smbpass 0 0
\r\n//pastis/share /home/clark/share smbfs user,noauto,credentials=/home/clark/.smbpass 0 0\r\n

\r\n\r\n

\r\nLes répertoires sont ensuite montés au démarrage de ma session grâce au script /home/clark/.config/autostart/smbmount suivant :\r\n

\r\n\r\n

\r\n#!/bin/bash
\r\nmount /home/clark/server
\r\nmount /home/clark/share
\r\n

\r\n\r\n
